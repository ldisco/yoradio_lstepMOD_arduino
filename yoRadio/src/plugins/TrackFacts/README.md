# TrackFacts Plugin

Track Facts Plugin — система для получения и отображения интересных фактов о композициях, которые играют на радио.

## Возможности

- 📊 Автоматическое получение информации о треках через различные API
- 🌍 Поддержка нескольких источников фактов:
  - **MusicBrainz** (бесплатно, без ключа)
  - **Last.fm** (бесплатно с API ключом)
  - Возможность добавления собственных источников
- 🔄 Циклическое отображение названия трека и факта о нём
- 🎯 Бегущая строка для длинных названий
- 🌐 Локализация: Английский и Русский языки
- ⚡ Легкая интеграция без изменения основного кода

## Архитектура

### Серверная часть (C++)

#### `TrackFacts.h` и `TrackFacts.cpp`
Основной плагин, который:
1. Подписывается на события изменения трека (`on_track_change`)
2. Проверяет метаданные трека каждую секунду (`on_ticker`)
3. Запрашивает факты через различные API через 7 секунд после определения трека
4. Отправляет факты на веб-интерфейс через `/api/current-fact` endpoint

**Файлы:**
- `src/src/plugins/TrackFacts/TrackFacts.h` — заголовок
- `src/src/plugins/TrackFacts/TrackFacts.cpp` — реализация

### Клиентская часть (JavaScript)

#### `facts.js`
Веб-интерфейс, который:
1. Получает факты с сервера
2. Управляет циклом отображения: название (7 сек) → факт (10 сек) → название
3. Реализует бегущую строку для длинных названий
4. Поддерживает локализацию (EN/RU)

**Файл:** `data/www/facts.js`

## Цикл работы

```
Трек определён (7 сек на определение)
         ↓
Плагин запрашивает факт через API
         ↓
Факт загружен в памяти
         ↓
[JavaScript цикл]
  ├─ Показать название трека (7 сек)
  ├─ Показать факт про трек (10 сек)
  └─ Повторить...
```

## Использование

### 1. Автоматическая загрузка

Плагин автоматически загружается при инициализации системы. Убедитесь, что `TrackFacts` инстанцируется в `main.cpp`:

```cpp
TrackFacts trackFactsPlugin;  // Регистрируется автоматически в конструкторе
```

### 2. Выбор источника фактов

В будущих версиях можно будет переключать источники через веб-интерфейс:

```cpp
// Использовать MusicBrainz (по умолчанию)
trackFactsPlugin.setFactSource(FactSource::MUSICBRAINZ);

// Или Last.fm с API ключом
trackFactsPlugin.setFactSource(FactSource::LASTFM);
trackFactsPlugin.setLastFmKey("your_api_key_here");
```

### 3. API Endpoints

#### GET `/api/current-fact`
Получить текущий факт и название трека

**Ответ:**
```json
{
  "fact": "Интересный факт про композицию...",
  "title": "Название трека - Исполнитель"
}
```

## Источники фактов

### MusicBrainz
- **Тип:** Открытая БД музыки
- **API:** Бесплатный
- **Лимиты:** 1 запрос в секунду
- **Преимущества:** Никакого ключа не нужно

### Last.fm
- **Тип:** Социальная сеть для музыки
- **API:** Бесплатный с регистрацией
- **Лимиты:** 200 запросов в сек (для API)
- **Преимущества:** Много информации о треках и исполнителях
